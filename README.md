# Michelin Explorer

A fast, worldwide directory of restaurant entities with a Michelin star award recorded in Wikidata. It provides country and city directories, restaurant details, and a responsive map without hidden recovery or inferred metadata.

Live site: <https://michelin.thefrenchartist.dev/>

## Experience

- Worldwide overview ranked by the number of sourced restaurant records
- Country pages with canonical-city directories and visibly unassigned records
- Complete alphabetical city pages and global restaurant search
- Responsive worldwide and city maps with an equivalent accessible list
- Explicit Wikidata provenance, missing-field states, and methodology

## Data

The dataset is generated by `scripts/generate_restaurants_csv.py` into `public/data/restaurants.csv`, with provenance and aggregate counts in `public/data/metadata.json`.

Use `make refresh` to create the CSV only when it does not already exist. Use `make data` to force a fresh download.

The generator queries Wikidata for every item that:

- has `award received` (`P166`) = `Michelin star` (`Q20824563`)
- is a `restaurant` (`Q11707`) or one of its subclasses
- may optionally have a `Michelin Restaurants ID` (`P4160`)

The auditable query lives in `scripts/wikidata_restaurants.sparql`. Location ancestry is resolved through the Wikidata API so the export can identify the nearest canonical city without inferring one from an address or external guide URL.

The refresh writes `public/data/restaurants.csv` and `public/data/metadata.json` only after the complete result passes validation. If Wikidata or validation fails, the previous public snapshot is left untouched.

Source contract and limits:

- Wikidata is the sole source for restaurant membership and metadata. There are no Wikipedia, Michelin-page, Algolia, geocoding, or generated-description fallbacks.
- Missing country, city, address, coordinates, website, cuisine, description, and Michelin ID values stay missing.
- `P166 = Q20824563` records that a Michelin star award exists; it does not reliably provide a current 1/2/3-star tier or prove that the restaurant is still operating.
- A canonical city is the nearest `P131` location whose type descends from `city` (`Q515`) after boroughs, wards, city districts, and exact neighborhood classifications are rejected. Resolution continues through rejected sub-city entities to a qualifying parent city.
- The excluded Wikidata class IDs are published in `metadata.json`. A sub-city classification wins over a simultaneous city-like classification; if no qualifying parent exists, the city stays missing.
- The CSV is a dated snapshot until `make data` is run again.

The export is validated so every returned Wikidata restaurant becomes exactly one row with a stable ID. Coordinates are range-checked, duplicate public IDs are rejected, and files are replaced only after the complete refresh succeeds.

## Routes

- `#/` — worldwide overview
- `#/countries/:countrySlug` — country and canonical-city directory
- `#/countries/:countrySlug/cities/:citySlug` — complete city restaurant list
- `#/restaurants` and `#/restaurants/:restaurantId` — global directory and details
- `#/map?country=&city=` — linkable map scope
- `#/about` — methodology, provenance, and limitations

## Commands

```bash
make up
make kill
make refresh
make test
make build
make deploy
```

## Development

```bash
npm install
make data
make up
```

Use `make kill` to stop the background development server. The app uses `HashRouter`, so deep links such as `#/restaurants` and `#/map` work on GitHub Pages.

Generator fixtures live in `scripts/test_generate_restaurants_csv.py`; React and data tests live alongside the source under `src`. `make test` runs both suites.

## Deployment

`make deploy` runs a fresh production build and publishes `dist` with the `gh-pages` package while preserving the `michelin.thefrenchartist.dev` custom domain.
